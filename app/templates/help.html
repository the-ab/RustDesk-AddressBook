Überblick

RustDesk AddressBook ist ein eigenes Web-Adressbuch für selbstgehostete RustDesk-Installationen. Es ersetzt nicht RustDesk Server Pro, bietet aber zentrale Geräteverwaltung, Gruppen, Suche, direkte RustDesk-Links, Import/Export, Backups, 2FA und Live-Online-Status über hbbs.

Geräteverwaltung
Name, RustDesk-ID, Passwort, Gruppe, Kunde, Standort, Gerätetyp, Tags und Notizen.
Live-Status
Online/Offline-Abfrage über RustDesk hbbs OnlineRequest/OnlineResponse.
Import / Export
CSV, RustDesk-Serverdatenbank, gemountete DB und SSH-Snapshot-Import.
Sicherheit
Admin-Login, 2FA, Recovery-Codes, Auditlog, Brute-Force-Sperre und verschlüsselte Backups.

Installation und Update

Neuinstallation mit Installationsscript

cd /opt
wget https://dl.ab-xnet.de/rustdesk-addressbook-v0517.zip
unzip rustdesk-addressbook-v0517.zip
cd rustdesk-addressbook
chmod +x scripts/install.sh scripts/update.sh
./scripts/install.sh

Das Script fragt HTTPS-Port, optionalen HTTP-Port, Datenverzeichnis, Backup-Verzeichnis, Brute-Force-Werte, Update-URL und optionalen read-only RustDesk-DB-Mount ab.

Die gesetzten Werte werden in .env gespeichert. Beim erneuten Aufruf werden diese Werte als Default vorgeschlagen.

Update einer bestehenden Installation

Variante A: Update-ZIP manuell in den vorhandenen Ordner updates/ laden:

cd /opt/rustdesk-addressbook
wget https://dl.ab-xnet.de/rustdesk-addressbook-update-flat-v0517.zip -O updates/rustdesk-addressbook-update-flat-v0517.zip
./scripts/update.sh

Variante B: automatischer Online-Check und Download:

cd /opt/rustdesk-addressbook
./scripts/update.sh

Das Updatescript erkennt lokale ZIPs in updates/. Ist lokal nichts Neueres vorhanden, prüft es online unter RAB_UPDATE_BASE_URL, zeigt bei verfügbarer Version die Änderungen an und fragt nach Download und Installation. Es erstellt automatisch eine Pre-Update-Sicherung und entfernt bei Bedarf einen alten Container mit demselben Namen.

Release-Notizen für Online-Updates

rustdesk-addressbook-update-flat-v0517.zip
- Änderung 1
- Änderung 2

Diese Zeilen können direkt in latest.txt unter der ZIP-Datei stehen. Alternativ kann eine separate Datei wie rustdesk-addressbook-update-flat-v0517.txt oder release-notes-v0517.txt im Download-Verzeichnis liegen. Die WebUI zeigt diese Änderungen unter Einstellungen → Update-Check an.

WebUI öffnen

https://SERVER-IP:5443
HTTP ist standardmäßig deaktiviert. Bei HTTPS ohne eigenes Zertifikat wird automatisch ein selbstsigniertes Zertifikat erzeugt. Die Browser-Warnung ist in diesem Fall normal.

Ersteinrichtung

  1. WebUI öffnen.
  2. Admin-Benutzername und Passwort erstellen.
  3. Danach unter Einstellungen → 2FA optional TOTP aktivieren.
  4. Recovery-Codes offline speichern.
  5. Unter Einstellungen → Online-Status hbbs-Host und Port konfigurieren.
Sichere regelmäßig den Ordner data/. Darin liegen Datenbank, Schlüsselmaterial, Logs und SSH-Keys. Für einen vollständigen Restore erstelle zusätzlich regelmäßig ein verschlüsseltes .rabfull-Vollbackup.

Geräte

Unter Geräte verwaltest du alle RustDesk-Ziele.

  • Verbinden: öffnet rustdesk://ID. Wenn ein Passwort hinterlegt ist, wird es erst beim Klick serverseitig entschlüsselt.
  • Auge-Symbol: zeigt ein gespeichertes Gerätepasswort nur auf Anfrage an.
  • Favorit: markiert wichtige Geräte für Dashboard und Sortierung.
  • Online: kann manuell gesetzt werden oder per hbbs Live-Abfrage aktualisiert werden.
  • Sortierung: Online-Geräte stehen standardmäßig oben.

Gruppen

Gruppen dienen zur Strukturierung nach Kunde, Standort, Gerätetyp oder Funktion. Jede Gruppe hat Name, Farbe und Icon.

  • Icons werden per Dropdown mit Vorschau gewählt.
  • Bestehende Gruppen können bearbeitet werden.
  • Gruppen mit Geräten können nicht gelöscht werden, solange Geräte zugeordnet sind.

Online-Status über hbbs

Der Live-Status wird über den RustDesk-ID-Server abgefragt. Die funktionierende Standardeinstellung ist normalerweise TCP-Port 21115.

Container-Verbindung testen

docker exec -it rustdesk-addressbook python - <<'PY'
import socket
host = "DEIN-RUSTDESK-SERVER"
port = 21115
s = socket.create_connection((host, port), timeout=3)
print("TCP Verbindung OK")
s.close()
PY

WebUI-Einstellungen

  • Statusquelle: hbbs Live-Abfrage
  • hbbs Host: IP oder DNS-Name deines RustDesk-ID-Servers
  • hbbs Port: meistens 21115
  • Automatik: Intervall in Minuten oder Stunden möglich, solange die WebUI geöffnet ist
Die hbbs-Abfrage nutzt das RustDesk-Protokoll, aber keine offiziell dokumentierte Web-API. Nach RustDesk-Serverupdates sollte die Funktion getestet werden.

Import / Export

Die Seite Import / Export bündelt alle Datenübernahmen.

CSV-Import

Unterstützte Spaltennamen:

name,rustdesk_id,password,customer,location,os,tags,notes,favorite,online,group

CSV-Export

  • Export ohne Passwörter: empfohlen für normale Dokumentation.
  • Export mit Passwörtern: nur für gesicherte Umgebungen verwenden.

RustDesk-Serverdatenbank per Upload

Akzeptiert einzelne SQLite-Dateien oder ZIP-Dateien mit:

db_v2.sqlite3
db_v2.sqlite3-wal
db_v2.sqlite3-shm

Import aus gemounteter DB

Expertenoption, wenn die RustDesk-DB read-only in den Container eingebunden ist.

volumes:
  - /docker_data/rustdesk:/rustdesk-server:ro

environment:
  RUSTDESK_SERVER_DB: /rustdesk-server/db_v2.sqlite3

RustDesk SSH-Import

Der empfohlene Weg für getrennte Server: Das Adressbuch holt per SSH einen konsistenten SQLite-Snapshot vom RustDesk-Server. Der SSH-Key wird auf ein einziges Export-Kommando beschränkt.

1. RustDesk-Server vorbereiten

apt update
apt install sqlite3 openssh-server acl
adduser --disabled-password --gecos "" rab-import

2. Export-Script erstellen

cat > /usr/local/sbin/rab-rustdesk-db-export <<'EOSCRIPT'
#!/bin/bash
set -euo pipefail

DB="/docker_data/rustdesk/db_v2.sqlite3"
TMP="$(mktemp /tmp/rustdesk-db-export.XXXXXX.sqlite3)"

cleanup() {
  rm -f "$TMP"
}
trap cleanup EXIT

if [ ! -r "$DB" ]; then
  echo "ERROR: RustDesk DB not readable: $DB" >&2
  exit 1
fi

sqlite3 "$DB" ".backup '$TMP'"
cat "$TMP"
EOSCRIPT
chmod 755 /usr/local/sbin/rab-rustdesk-db-export

3. Leserechte für den Import-Benutzer

setfacl -m u:rab-import:rx /docker_data
setfacl -m u:rab-import:rx /docker_data/rustdesk
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-wal 2>/dev/null || true
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-shm 2>/dev/null || true

4. SSH-Key auf dem Adressbuch-Server erzeugen

cd /opt/rustdesk-addressbook
mkdir -p data/ssh
ssh-keygen -t ed25519 -f data/ssh/rustdesk_import_ed25519 -N ""
chmod 600 data/ssh/rustdesk_import_ed25519
cat data/ssh/rustdesk_import_ed25519.pub

5. Public Key auf dem RustDesk-Server einschränken

mkdir -p /home/rab-import/.ssh
nano /home/rab-import/.ssh/authorized_keys

In authorized_keys eine Zeile nach diesem Muster eintragen:

restrict,no-pty,no-agent-forwarding,no-X11-forwarding,no-port-forwarding,command="/usr/local/sbin/rab-rustdesk-db-export" ssh-ed25519 AAAA...
chown -R rab-import:rab-import /home/rab-import/.ssh
chmod 700 /home/rab-import/.ssh
chmod 600 /home/rab-import/.ssh/authorized_keys

6. Manueller SSH-Test

ssh -T \
  -i ./data/ssh/rustdesk_import_ed25519 \
  -o IdentitiesOnly=yes \
  -o BatchMode=yes \
  rab-import@RUSTDESK-SERVER-IP \
  > /tmp/rustdesk-db-snapshot.sqlite3

file /tmp/rustdesk-db-snapshot.sqlite3
sqlite3 /tmp/rustdesk-db-snapshot.sqlite3 ".tables"
sqlite3 /tmp/rustdesk-db-snapshot.sqlite3 "PRAGMA integrity_check;"
sqlite3 /tmp/rustdesk-db-snapshot.sqlite3 "SELECT COUNT(*) FROM peer;"

Die Meldung PTY allocation request failed ist bei eingeschränktem Key nicht kritisch. Mit ssh -T wird keine PTY angefordert.

7. WebUI konfigurieren

  • SSH Host: RustDesk-Server-IP oder DNS
  • SSH Benutzer: rab-import
  • Private-Key-Pfad: /data/ssh/rustdesk_import_ed25519
  • Remote-Kommando: leer lassen, wenn forced command in authorized_keys genutzt wird
  • SSH-Übertragung testen: prüft Dateiübertragung, SQLite-Header, Integrity-Check und Peer-Anzahl

Backup und Restore

Backups werden im Ordner backups/ abgelegt. Es gibt normale SQLite-Backups, verschlüsselte .rabenc-Datenbankbackups und verschlüsselte .rabfull-Vollbackups.

Empfohlen: verschlüsseltes Vollbackup

Backup-Seite öffnen
→ Verschlüsseltes Vollbackup erstellen
→ starkes Vollbackup-Passwort setzen
→ .rabfull-Datei extern sichern

Das Vollbackup enthält addressbook.db, config.json, data/ssh, data/certs und data/logs. Nach Restore eines Vollbackups den Container neu starten.

Hostseitige Sicherung

cd /opt
borg create /backup/rustdesk-addressbook::rab-$(date +%F-%H%M) rustdesk-addressbook/data rustdesk-addressbook/backups
Ein unverschlüsseltes Backup enthält die komplette AddressBook-Datenbank. Für externe Ablage bevorzugt .rabfull verwenden; .rabenc enthält nur die Datenbank und benötigt zusätzlich die passende config.json.

Sicherheit

  • 2FA/TOTP: Aktivierung über Einstellungen, Recovery-Codes nur einmal sichtbar.
  • Brute-Force-Sperre: Fehlversuche und Zeitraum unter Einstellungen konfigurierbar.
  • Auditlog: Login- und 2FA-Ereignisse unter Sicherheit.
  • Passwörter: Gerätepasswörter werden verschlüsselt gespeichert und nur bei Bedarf entschlüsselt.
  • HTTPS: direkt im Container möglich; Reverse Proxy mit gültigem Zertifikat ist weiterhin empfohlen.

Eigene TLS-Zertifikate

mkdir -p data/certs
cp fullchain.pem data/certs/addressbook.crt
cp privkey.pem data/certs/addressbook.key
chmod 600 data/certs/addressbook.key
docker compose up -d --force-recreate

fail2ban / CrowdSec

Fehlgeschlagene Logins werden zusätzlich in data/logs/auth.log geschrieben. Relevante Zeilen enthalten RAB_AUTH_FAIL.

Log prüfen

tail -f /opt/rustdesk-addressbook/data/logs/auth.log

fail2ban-Beispiel

cp contrib/fail2ban/filter.d/rustdesk-addressbook.conf /etc/fail2ban/filter.d/
cp contrib/fail2ban/jail.d/rustdesk-addressbook.local.example /etc/fail2ban/jail.d/rustdesk-addressbook.local
systemctl restart fail2ban
fail2ban-client status rustdesk-addressbook

Wenn du CrowdSec als Banaction nutzt, kann fail2ban weiterhin den Treffer erkennen und CrowdSec für die Sperre verwenden.

Fehlerdiagnose

Containerlogs

docker compose logs -f --tail=200

Version prüfen

docker exec -it rustdesk-addressbook grep -n "APP_VERSION" /app/app/config.py

Datenbank prüfen

sqlite3 data/addressbook.db "PRAGMA integrity_check;"

SSH-Key-Rechte prüfen

ls -l data/ssh/rustdesk_import_ed25519
docker exec -it rustdesk-addressbook ls -l /data/ssh/rustdesk_import_ed25519

hbbs-Port prüfen

docker exec -it rustdesk-addressbook python - <<'PY'
import socket
socket.create_connection(("DEIN-RUSTDESK-SERVER", 21115), timeout=3).close()
print("OK")
PY
