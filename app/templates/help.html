Overview

RustDesk AddressBook is an independent web address book for self-hosted RustDesk environments. It does not replace RustDesk Server Pro, but provides central device management, groups, search, RustDesk links, import/export, backups, 2FA and live online status via hbbs.

Device management
Name, RustDesk ID, password, group, customer, location, OS/device type, tags and notes.
Live status
Online/offline checks using the RustDesk hbbs OnlineRequest/OnlineResponse flow.
Import / Export
CSV, RustDesk server DB upload, mounted DB import and SSH snapshot import.
Security
Admin login, 2FA, recovery codes, audit log, brute-force lockout and encrypted backups.

Installation and update

Fresh installation

cd /opt
wget https://dl.ab-xnet.de/rustdesk-addressbook-v0520.zip
unzip rustdesk-addressbook-v0520.zip
cd rustdesk-addressbook
chmod +x scripts/install.sh scripts/update.sh
./scripts/install.sh

The installer asks for timezone, container/image name, data and backup directories, HTTPS port, optional HTTP port, certificate names, reverse-proxy trust, optional read-only RustDesk DB mount, brute-force values and the update download base URL.

The values are written to .env. When you run ./scripts/install.sh again, the existing .env values are used as defaults.

Manual update with ZIP in updates/

cd /opt/rustdesk-addressbook
wget https://dl.ab-xnet.de/rustdesk-addressbook-update-flat-v0520.zip -O updates/rustdesk-addressbook-update-flat-v0520.zip
./scripts/update.sh

Automatic online update check

cd /opt/rustdesk-addressbook
./scripts/update.sh

The update script first checks local ZIP files in updates/. If none is newer, it reads latest.txt from RAB_UPDATE_BASE_URL, shows release notes and asks before downloading and installing.

Web UI

https://SERVER-IP:5443
HTTP is disabled by default. If no own certificate is configured, a self-signed certificate is generated and the browser warning is expected.

First setup

  1. Open the Web UI.
  2. Create the admin username and password.
  3. Configure appearance and language under Settings → Display & language.
  4. Enable 2FA under Settings → 2FA / Authenticator and save recovery codes offline.
  5. Configure hbbs host/port under Settings → Online status.
  6. Create an encrypted full backup after the base setup.
The data/ directory contains the database, config, logs, SSH keys and certificates. Use encrypted .rabfull full backups for complete disaster recovery.

Dashboard and device views

The dashboard shows device counters, online devices, favorites and recently changed devices. The Live status hbbs button can refresh all online states.

  • Card view: current detailed layout.
  • List view: compact rows for many devices.
  • Small icon view: very compact overview.

The selected view is stored in the session and used on Dashboard and Devices.

Devices

  • Connect: opens rustdesk://ID. If a password is stored, it is decrypted server-side only on click.
  • Eye icon: shows the saved device password only on explicit request.
  • Favorite: marks important devices for dashboard and sorting.
  • Online: can be set manually or refreshed using hbbs live status.
  • Sort: online devices are shown first by default.
  • Search/filter: supports name, ID, customer, location, OS/device type, tags and notes.

Groups

Groups help structure devices by customer, site, device type or purpose. Each group has a name, color and icon.

  • Icons are selected from a dropdown with preview.
  • Existing groups can be edited.
  • Groups that still contain devices cannot be deleted until the devices are moved or deleted.

Online status via hbbs

The live status is queried from the RustDesk ID server. In the tested setup the working port is usually TCP 21115.

docker exec -it rustdesk-addressbook python - <<'PY'
import socket
host = "YOUR-RUSTDESK-SERVER"
port = 21115
s = socket.create_connection((host, port), timeout=3)
print("TCP connection OK")
s.close()
PY
  • Status source: hbbs live query.
  • hbbs host: IP or DNS name of the RustDesk ID server.
  • hbbs port: usually 21115.
  • Automatic check: minute/hour interval while the Web UI is open.
  • Last result: shown on Dashboard and Devices next to the hbbs button.
The hbbs query uses RustDesk protocol messages, but it is not an officially documented Web API. Retest after RustDesk server upgrades.

environment:

RustDesk SSH import

Recommended for separate servers: AddressBook pulls a consistent SQLite snapshot from the RustDesk server via SSH. The SSH key should be restricted to one forced command.

1. Prepare RustDesk server

apt update
apt install sqlite3 openssh-server acl
adduser --disabled-password --gecos "" rab-import

2. Create export script

cat > /usr/local/sbin/rab-rustdesk-db-export <<'EOSCRIPT'
#!/bin/bash
set -euo pipefail
DB="/docker_data/rustdesk/db_v2.sqlite3"
TMP="$(mktemp /tmp/rustdesk-db-export.XXXXXX.sqlite3)"
cleanup() { rm -f "$TMP"; }
trap cleanup EXIT
if [ ! -r "$DB" ]; then
  echo "ERROR: RustDesk DB not readable: $DB" >&2
  exit 1
fi
sqlite3 "$DB" ".backup '$TMP'"
cat "$TMP"
EOSCRIPT
chmod 755 /usr/local/sbin/rab-rustdesk-db-export

3. Permissions

setfacl -m u:rab-import:rx /docker_data
setfacl -m u:rab-import:rx /docker_data/rustdesk
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-wal 2>/dev/null || true
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-shm 2>/dev/null || true

4. Restricted SSH key

restrict,no-pty,no-agent-forwarding,no-X11-forwarding,no-port-forwarding,command="/usr/local/sbin/rab-rustdesk-db-export" ssh-ed25519 AAAA...

5. Local key path

mkdir -p data/ssh
cp rustdesk_import_ed25519 data/ssh/
chmod 600 data/ssh/rustdesk_import_ed25519

In the Web UI use /data/ssh/rustdesk_import_ed25519. The SSH test reports connection, transfer size, SQLite header, integrity check, peer table and peer count.

Backup / restore

  • DB backup: quick SQLite database copy.
  • Encrypted DB backup: encrypted database backup.
  • Encrypted full backup .rabfull: recommended for migration and disaster recovery. Includes addressbook.db, config.json, data/ssh/, data/certs/, data/logs/ and a manifest.
Store the full-backup password outside the server. After restoring a full backup, restart the container so the restored config.json is reloaded.

Settings

The settings page uses a left navigation and a detail area.

  • Display & language: light/dark mode and German/English UI language.
  • Admin account: change admin username and password.
  • 2FA: enable TOTP, manage recovery codes and disable 2FA with password plus OTP/recovery code.
  • Device types: customize OS/device type presets.
  • Online status: hbbs host, port, timeout, batch size and automatic checks.
  • Brute-force lockout: failed attempts and time window.
  • Update check: automatic online checks and interval.

Update check and release notes

The Web UI checks latest.txt below RAB_UPDATE_BASE_URL. If an update is available, a badge appears next to Admin/Logout and links to Settings → Update check.

rustdesk-addressbook-update-flat-v0520.zip
[de]
- Deutsche Änderung 1
[en]
- English change 1

If no language sections exist, all lines below the ZIP filename are shown as release notes.

Security

  • Use HTTPS. Reverse proxy TLS is recommended for public access.
  • Enable 2FA and store recovery codes offline.
  • Use encrypted full backups for anything containing config.json, SSH keys or TLS keys.
  • Check the Security page for 2FA, logs, permissions, secure cookies, HSTS and backup status.
  • Do not run the address book on the RustDesk server if you want stronger separation.

fail2ban / CrowdSec

Failed login attempts are written to:

data/logs/auth.log

Failed authentication lines contain:

RAB_AUTH_FAIL

Example files are included under contrib/fail2ban/.

Troubleshooting

docker compose ps
docker compose logs -f
docker exec -it rustdesk-addressbook grep -n "{{ app_version }}" /app/app/config.py

  • Update conflict: current update script removes an existing container with the same name before recreating it.
  • SSH import: use the built-in SSH test first and check transfer size, integrity and peer count.
  • Online status: test TCP access to hbbs port 21115.

RustDesk AddressBook ist ein eigenes Web-Adressbuch für selbstgehostete RustDesk-Installationen. Es ersetzt nicht RustDesk Server Pro, bietet aber zentrale Geräteverwaltung, Gruppen, Suche, direkte RustDesk-Links, Import/Export, Backups, 2FA und Live-Online-Status über hbbs.

Geräteverwaltung
Name, RustDesk-ID, Passwort, Gruppe, Kunde, Standort, Gerätetyp, Tags und Notizen.
Live-Status
Online/Offline-Abfrage über RustDesk hbbs OnlineRequest/OnlineResponse.
Import / Export
CSV, RustDesk-Serverdatenbank, gemountete DB und SSH-Snapshot-Import.
Sicherheit
Admin-Login, 2FA, Recovery-Codes, Auditlog, Brute-Force-Sperre und verschlüsselte Backups.

Installation und Update

Neuinstallation mit Installationsscript

cd /opt
wget https://dl.ab-xnet.de/rustdesk-addressbook-v0520.zip
unzip rustdesk-addressbook-v0520.zip
cd rustdesk-addressbook
chmod +x scripts/install.sh scripts/update.sh
./scripts/install.sh

Das Script fragt HTTPS-Port, optionalen HTTP-Port, Datenverzeichnis, Backup-Verzeichnis, Brute-Force-Werte, Update-URL und optionalen read-only RustDesk-DB-Mount ab.

Die gesetzten Werte werden in .env gespeichert. Beim erneuten Aufruf werden diese Werte als Default vorgeschlagen.

Update-ZIP manuell ablegen

cd /opt/rustdesk-addressbook
wget https://dl.ab-xnet.de/rustdesk-addressbook-update-flat-v0520.zip -O updates/rustdesk-addressbook-update-flat-v0520.zip
./scripts/update.sh

Automatischer Online-Check

cd /opt/rustdesk-addressbook
./scripts/update.sh

Das Updatescript erkennt lokale ZIPs in updates/. Ist lokal nichts Neueres vorhanden, prüft es online unter RAB_UPDATE_BASE_URL, zeigt Änderungen an und fragt nach Download und Installation.

WebUI öffnen

https://SERVER-IP:5443
HTTP ist standardmäßig deaktiviert. Bei HTTPS ohne eigenes Zertifikat wird automatisch ein selbstsigniertes Zertifikat erzeugt. Die Browser-Warnung ist in diesem Fall normal.

Ersteinrichtung

  1. WebUI öffnen.
  2. Admin-Benutzername und Passwort erstellen.
  3. Unter Einstellungen → Darstellung & Sprache Theme und Sprache wählen.
  4. Unter Einstellungen → 2FA optional TOTP aktivieren.
  5. Recovery-Codes offline speichern.
  6. Unter Einstellungen → Online-Status hbbs-Host und Port konfigurieren.
  7. Danach ein verschlüsseltes Vollbackup erstellen.
Sichere regelmäßig data/ oder erstelle ein verschlüsseltes .rabfull-Vollbackup. Darin liegen Datenbank, Schlüsselmaterial, Logs, SSH-Keys und Zertifikate.

Dashboard und Geräteansichten

Das Dashboard zeigt Zähler, Online-Geräte, Favoriten und zuletzt geänderte Geräte. Der Button Live-Status hbbs aktualisiert die Online-Zustände.

  • Kartenansicht: aktuelle detaillierte Darstellung.
  • Listenansicht: kompakte Zeilen für viele Geräte.
  • Kleine Symbole: sehr kompakte Übersicht.

Die gewählte Ansicht wird in der Session gespeichert und auf Dashboard und Geräte-Seite verwendet.

Geräte

  • Verbinden: öffnet rustdesk://ID. Wenn ein Passwort hinterlegt ist, wird es erst beim Klick serverseitig entschlüsselt.
  • Auge-Symbol: zeigt ein gespeichertes Gerätepasswort nur auf Anfrage an.
  • Favorit: markiert wichtige Geräte für Dashboard und Sortierung.
  • Online: kann manuell gesetzt oder per hbbs Live-Abfrage aktualisiert werden.
  • Sortierung: Online-Geräte stehen standardmäßig oben.
  • Suche/Filter: durchsucht Name, ID, Kunde, Standort, Gerätetyp, Tags und Notizen.

Gruppen

Gruppen dienen zur Strukturierung nach Kunde, Standort, Gerätetyp oder Funktion. Jede Gruppe hat Name, Farbe und Icon.

  • Icons werden per Dropdown mit Vorschau gewählt.
  • Bestehende Gruppen können bearbeitet werden.
  • Gruppen mit Geräten können nicht gelöscht werden, solange Geräte zugeordnet sind.

Online-Status über hbbs

Der Live-Status wird über den RustDesk-ID-Server abgefragt. Die funktionierende Standardeinstellung ist normalerweise TCP-Port 21115.

docker exec -it rustdesk-addressbook python - <<'PY'
import socket
host = "DEIN-RUSTDESK-SERVER"
port = 21115
s = socket.create_connection((host, port), timeout=3)
print("TCP Verbindung OK")
s.close()
PY
  • Statusquelle: hbbs Live-Abfrage
  • hbbs Host: IP oder DNS-Name deines RustDesk-ID-Servers
  • hbbs Port: meistens 21115
  • Automatik: Intervall in Minuten oder Stunden möglich, solange die WebUI geöffnet ist
  • Letztes Ergebnis: wird auf Dashboard und Geräte-Seite neben dem hbbs-Button angezeigt
Die hbbs-Abfrage nutzt das RustDesk-Protokoll, aber keine offiziell dokumentierte Web-API. Nach RustDesk-Serverupdates sollte die Funktion getestet werden.

RustDesk SSH-Import

Der empfohlene Weg für getrennte Server: Das Adressbuch holt per SSH einen konsistenten SQLite-Snapshot vom RustDesk-Server. Der SSH-Key wird auf ein einziges Export-Kommando beschränkt.

1. RustDesk-Server vorbereiten

apt update
apt install sqlite3 openssh-server acl
adduser --disabled-password --gecos "" rab-import

2. Export-Script erstellen

cat > /usr/local/sbin/rab-rustdesk-db-export <<'EOSCRIPT'
#!/bin/bash
set -euo pipefail
DB="/docker_data/rustdesk/db_v2.sqlite3"
TMP="$(mktemp /tmp/rustdesk-db-export.XXXXXX.sqlite3)"
cleanup() { rm -f "$TMP"; }
trap cleanup EXIT
if [ ! -r "$DB" ]; then
  echo "ERROR: RustDesk DB not readable: $DB" >&2
  exit 1
fi
sqlite3 "$DB" ".backup '$TMP'"
cat "$TMP"
EOSCRIPT
chmod 755 /usr/local/sbin/rab-rustdesk-db-export

3. Leserechte setzen

setfacl -m u:rab-import:rx /docker_data
setfacl -m u:rab-import:rx /docker_data/rustdesk
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-wal 2>/dev/null || true
setfacl -m u:rab-import:r /docker_data/rustdesk/db_v2.sqlite3-shm 2>/dev/null || true

4. SSH-Key einschränken

restrict,no-pty,no-agent-forwarding,no-X11-forwarding,no-port-forwarding,command="/usr/local/sbin/rab-rustdesk-db-export" ssh-ed25519 AAAA...

5. Key im Adressbuch ablegen

mkdir -p data/ssh
cp rustdesk_import_ed25519 data/ssh/
chmod 600 data/ssh/rustdesk_import_ed25519

In der WebUI wird dann /data/ssh/rustdesk_import_ed25519 verwendet. Der SSH-Test prüft Verbindung, übertragene Bytes, SQLite-Header, Integrity-Check, peer-Tabelle und Anzahl der Peers.

Backup / Restore

  • DB-Backup: schnelle Kopie der SQLite-Datenbank.
  • Verschlüsseltes DB-Backup: verschlüsselte Sicherung der Datenbank.
  • Verschlüsseltes Vollbackup .rabfull: empfohlen für Umzug und Totalausfall. Enthält addressbook.db, config.json, data/ssh/, data/certs/, data/logs/ und Manifest.
Das Vollbackup-Passwort getrennt vom Server speichern. Nach Restore eines Vollbackups den Container neu starten, damit die wiederhergestellte config.json geladen wird.

Einstellungen

Die Einstellungen sind in linker Navigation und rechten Detailbereich aufgeteilt.

  • Darstellung & Sprache: heller/dunkler Modus und Deutsch/Englisch.
  • Admin-Konto: Admin-Benutzername und Passwort ändern.
  • 2FA: TOTP aktivieren, Recovery-Codes verwalten und 2FA mit Passwort plus OTP/Recovery-Code deaktivieren.
  • Gerätetypen: Betriebssystem-/Gerätetyp-Vorauswahl anpassen.
  • Online-Status: hbbs Host, Port, Timeout, Batchgröße und automatische Checks.
  • Brute-Force-Sperre: Fehlversuche und Zeitfenster.
  • Update-Check: automatische Online-Prüfung und Intervall.

Sicherheit

  • HTTPS verwenden. Für öffentlichen Zugriff ist ein Reverse Proxy mit TLS empfehlenswert.
  • 2FA aktivieren und Recovery-Codes offline speichern.
  • Backups mit config.json, SSH-Keys oder TLS-Keys nur verschlüsselt speichern.
  • Die Seite Sicherheit prüft 2FA, Logs, Berechtigungen, Secure Cookies, HSTS und Backup-Status.
  • Für stärkere Trennung das Adressbuch nicht direkt auf dem RustDesk-Server betreiben.

Fehlgeschlagene Loginversuche werden hier protokolliert:

data/logs/auth.log

RAB_AUTH_FAIL

Beispieldateien liegen unter contrib/fail2ban/.
